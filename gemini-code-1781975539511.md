# Functional Specification: AI-Driven Dynamic Team Allocation Engine (MVP)

## 1. Project Overview & Scope
A rapid prototype for an enterprise resource management application that dynamically builds project teams based on custom project workloads, goals, and historic collaboration data. 

Instead of rigid squad sizes, the system analyzes task volume to determine optimal team sizes and matches employees using multidimensional profile arrays and graph-based synergy scores.

**Time Constraint:** 8-Hour Hackathon/Demo MVP.
**Architecture Principle:** Monolithic JavaScript/TypeScript (Next.js App Router). No external databases, no Python. Everything runs in-memory using local JSON data.

---

## 2. Technical Stack Boundaries
- **Framework:** Next.js (React) + Tailwind CSS
- **Graph Framework:** Pure JavaScript Graph Logic (Matrix Map)
- **Data Store:** `src/data/dataset.json` (Static JSON file)
- **Visualization:** D3.js or Cytoscape.js for node-link web graph rendering

---

## 3. Data Schema (`src/data/dataset.json`)
The application relies on a mock dataset containing explicit metrics:

### Employees Array
Each employee object contains:
- `id` (string)
- `name` (string)
- `age` (number)
- `gender` (string)
- `job_title` (string)
- `years_of_experience` (number)
- `knowledge_tags` (array of strings)
- `skill_vector` (array of 4 numbers [Tech, Management, Design, Operations], normalized 0.0 to 1.0)

### Synergy Matrix Map
A flat dictionary where keys are combined employee IDs (e.g., `EMP_001_EMP_002`) mapping historical workspace data:
- `past_collaborations` (number of shared tasks)
- `sentiment_score` (float from -1.0 to 1.0 representing collaboration friction or flow)

---

## 4. Core Core Logic Pipelines (The Matching Algorithm)

### A. Dynamic Team Sizing & Vector Translation
- **Input:** `projectGoal` (string description) and `workloadFactor` (integer 1-10)
- **Action:** 1. Determine required team size dynamically: `Math.ceil(workloadFactor / 2) + 1`
  2. Parse or mock a 4-dimensional target vector based on the keywords inside `projectGoal`.

### B. Synergy & Skill Filtering
1. **Skill Match:** Calculate the Cosine Similarity between each employee's `skill_vector` and the target project vector.
2. **Synergy Scoring:** For any potential team combination, loop through all paired connections and pull their `sentiment_score` from the `synergy_matrix`.
3. **Selection:** Filter out the candidates that maximize both individual capability and average team sentiment, respecting age/experience balancing toggles if selected.

---

## 5. UI Layout Requirements
The dashboard interface is broken down into three logical horizontal modules:

- **Top Control Panel:** - Text area for Project Goals / File drop area.
  - Workload Slider (1 to 10).
  - "Run AI Allocation Engine" button.
- **Middle Visual Panel ("The Giraffe Web Graph"):**
  - An interactive D3 network node-link graph displaying all employees as nodes.
  - Links represent past collaboration channels. When allocation executes, the selected sub-graph links glow to highlight the chosen team cluster.
- **Bottom Recommendation Columns:**
  - Vertical columns organized by department categories (e.g., Engineering, Design, Product, Marketing).
  - The highest scoring allocated team members populate the tops of their respective columns alongside their personalized task assignments.
